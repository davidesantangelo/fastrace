Built for speed. Designed for clarity.

Every design decision optimized for performance

Non-Blocking I/O

Event-driven pipeline using poll() with monotonic timing for stable RTT measurements

TTL Batching

Groups probes per hop with single setsockopt call, minimizing syscall overhead

Async DNS

Dedicated thread for reverse lookups keeps the probe loop unblocked

Load-Balanced Paths

Tree-style visualization shows branching at ECMP routes with precise per-probe RTTs

Fully Tunable

Adjust concurrency, probes, delays, and timeouts on the fly via command-line flags

Hardened Parsing

Bounds-checked ICMP handling with strict resource cleanup and signal handling

Get started in seconds

One command to install, one command to trace

1 Install

# Clone and build
git clone https://github.com/davidesantangelo/fastrace
cd fastrace
make
sudo make install

2 Run

# Basic usage
sudo ./fastrace google.com

# Fast mode (no DNS)
sudo ./fastrace -n google.com

# Custom settings
sudo ./fastrace -m 20 -q 5 -c 8 example.com

Command-line options

-nDisable DNS lookups
-m <hops>Max hops (default: 30, max: 128)
-q <probes>Probes per hop (default: 3, max: 10)
-c <count>Concurrent TTLs (default: 6)
-d <µs>Inter-probe delay (default: 250µs)
-W <ms>Poll timeout (default: 2ms)
-t <ms>Hop timeout (default: 700ms)
-P <port>Base UDP port (default: 33434)
